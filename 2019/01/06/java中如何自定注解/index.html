java中如何自定注解

通过自定注解简化代码，提高效率

注解的原理

简单的说，可以吧java中注解比作一个“标记”，将某个注解@Controller加到类、方法等上面，来告诉要调用此类(方法)的类一些信息，比如要给某成员变量赋值，在调用时就会为这个方法附上注解中的值
注解分为java原生中的注解以及Spring容器中的注解

注解方法使用@intterface修饰，并在上面加上元注解(元注解的作用就是负责注解其他注解)@Retention @Target

@Retention
  • SOURCE 被编译器忽略
  • CLASS 注解将会被保留在Class文件中，但在运行时并不会被VM保留。这是默认行为，所有没有用Retention注解的注解，都会采用这种策略。
  • RUNTIME 保留至运行时。所以我们可以通过反射去获取注解信息。
@Target 作用域
  • CONSTRUCTOR:用于描述构造器
  • FIELD:用于描述域
  • LOCAL_VARIABLE:用于描述局部变量
  • METHOD:用于描述方法
  • PACKAGE:用于描述包
  • PARAMETER:用于描述参数
  • TYPE:用于描述类、接口(包括注解类型) 或enum声明
